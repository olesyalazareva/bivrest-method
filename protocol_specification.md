# Спецификация протокола BIVREST

**Версия:** 1.0

---

## 1. Формат лога

Каждая запись (строка) в логе представляет собой структурированный JSON-объект.

| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| `session_id` | string | Да | Уникальный идентификатор сессии |
| `round` | integer | Да | Номер раунда (начинается с 1) |
| `player_id` | string | Да | Идентификатор игрока |
| `avatar` | string | Да | Название роли игрока |
| `task` | string | Да | Задание, которое выполнял игрок |
| `action_text` | text | Да | Отыгрыш (что сказал или сделал игрок) |
| `votes_believe` | integer | Да | Количество голосов «Верю» |
| `votes_dont_believe` | integer | Да | Количество голосов «Не верю» |
| `verdict` | string | Да | Итоговый вердикт: «Верю» или «Не верю» |
| `tokens_lost` | integer | Нет | Количество потерянных ресурсов (если применимо) |
| `empty_chair` | boolean | Нет | Был ли выход из роли (true/false) |

---

## 2. Правила голосования

- **Кворум:** Минимальное количество голосующих — 3 человека.
- **Подсчёт:** Вердикт определяется простым большинством голосов.
- **Ничья:** При равном количестве голосов ведущий или фасилитатор имеет решающий голос.
- **Анонимность:** Голосование может быть как открытым, так и анонимным — определяется перед началом сессии.

---

## 3. Структура сессии

1. **Подготовка:** Определяется критерий голосования, назначаются роли участников.
2. **Проведение:** Последовательность раундов. В каждом раунде:
   - Игроку выдаётся задание
   - Игрок выполняет задание
   - Группа голосует
   - Результат фиксируется в логе
3. **Завершение:** Сессия завершается по решению фасилитатора или после выполнения заранее оговорённого числа раундов.

---

## 4. Пример валидной записи

```json
{
  "session_id": "S001",
  "round": 4,
  "player_id": "P03",
  "avatar": "Бухгалтер",
  "task": "Опиши момент полного подчинения",
  "action_text": "Я опускаю голову и шепчу: «У меня нет власти. Ты решаешь».",
  "votes_believe": 4,
  "votes_dont_believe": 1,
  "verdict": "Верю",
  "tokens_lost": 0,
  "empty_chair": false
}
5. Валидация данных
Проверка обязательных полей:

Все поля, отмеченные как «Обязательное», должны присутствовать.

session_id не может быть пустой строкой.

round — положительное целое число.

votes_believe и votes_dont_believe — целые неотрицательные числа.

verdict — строго одно из значений: «Верю» или «Не верю».

Обработка ошибок:

Если запись не проходит валидацию, она не включается в итоговый датасет, а помещается в отдельный файл ошибок с указанием причины.
