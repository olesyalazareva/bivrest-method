# BIVREST — метод игровой деконструкции сценариев

**Поведенческая верификация через ролевую игру и бинарное голосование**

**Проблема:** AI-инженеры и исследователи тратят месяцы и сотни тысяч долларов на ручную разметку данных для обучения моделей детекции лжи и распознавания эмоций. Готовых датасетов в открытом доступе почти нет, а те, что есть — синтетические или низкого качества.

**Что делает BIVREST:** Это готовый протокол, который **генерирует размеченные данные автоматически** через игру с механикой «Верю / Не верю». Каждое действие игрока получает объективную бинарную метку без участия эксперта.

**Результат:** Вы получаете качественный датасет с реальными поведенческими реакциями для обучения AI за **часы, а не месяцы**, и без ручной разметки.

## 🔥 Какую проблему решает BIVREST

| **Проблема** | **Традиционный подход** | **BIVREST** |
|:---|:---|:---|
| Где взять данные? | Нанять десятки людей для ручной разметки | Игровая сессия автоматически генерирует разметку |
| Субъективность | Разметчики ошибаются или имеют предвзятость | Бинарное голосование «Верю/Не верю» дает объективный результат |
| Стоимость | Тысячи долларов за размеченные датасеты | Минимальные затраты на организацию игры |
| Время | Месяцы на сбор и разметку | Данные собираются в реальном времени за одну сессию |

## 🧠 Как это работает

1. **Игровая сессия:** 2–6 человек играют в ролевую игру с аватарами. Каждый ход — действие от лица персонажа.

2. **Бинарное голосование:** После каждого действия группа голосует **«Верю»** или **«Не верю»**.

3. **Штраф за несоответствие роли:** Если большинство сказало «Не верю» — игрок теряет ресурс. Штрафуется **не правда или ложь**, а **фантазия, не соответствующая роли**. Это создает естественный стимул быть эмоционально достоверным.

4. **Пустой стул:** После двух «Не верю» подряд игрок выходит из роли и говорит от себя. AI получает эталонный пример искреннего ответа.

5. **Готовый датасет:** Каждое действие, голос и результат записываются в структурированный лог (CSV/JSON). На выходе — датасет с полями: `action`, `verdict` (Верю/Не верю), `player_id`, `avatar`.

## 🚀 Как использовать метод

1. Изучите полный протокол в [`protocol/BIVREST_method.md`](./protocol/BIVREST_method.md)
2. Организуйте пилотные сессии по протоколу
3. Собирайте логи действий и голосований «Верю / Не верю»
4. Аннотируйте данные по гайду в [`annotation-guide/README.md`](./annotation-guide/README.md)
5. Обучайте модель на этих данных
